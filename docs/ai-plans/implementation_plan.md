# 포트폴리오 웹사이트 구축 기획서 (가독성 중심형)

본 기획서는 `/docs` 디렉토리의 성과 문서들을 원천 데이터(Source Material)로 삼아, 채용 담당자 및 실무 면접관이 읽기 편한 **'기술 경력서 형태의 웹 포트폴리오'**를 구축하기 위한 계획입니다. 마크다운 파일을 있는 그대로 첨부하거나 렌더링하는 것이 아니라, 해당 내용들을 웹 포트폴리오 포맷에 맞게 UI로 재가공하여 구성합니다.

## 1. 기획 목표 (Goal)
- **가독성 극대화**: 화려한 스크롤 애니메이션이나 복잡한 효과를 배제하고 글 자체의 전달력에 집중합니다. 다크 모드는 별도로 지원하지 않으며, 깔끔한 라이트 모드 기반으로 워드프레스/블로그처럼 편안하게 읽히도록 합니다.
- **전문성 강조**: '결제 도메인의 신뢰성'이라는 서사를 중심으로 기술적 고민과 문제 해결 과정을 포트폴리오 포맷으로 재구성합니다.
- **이력서(Resume) 분리 운영**: 
  - 요약본 역할의 '1-Page PDF 영문 이력서' 및 '제출용 포트폴리오(PPT/PDF)'는 별도로 관리합니다. 
  - 본 웹사이트는 언제든 링크 하나로 접근 가능한 **'온라인 기술 포트폴리오'** 역할을 수행합니다.

---

## 2. 기술 스택 (Tech Stack)

사용자의 요청에 따라 Vercel 배포에 최적화된 스택으로 구성합니다. 플러그인은 실제 적용 전에 승인을 거칩니다.

- **프레임워크**: **Next.js (App Router)**
  - Vercel에 손쉽게 배포 가능하며, 컴포넌트 기반 아키텍처 및 SEO 최적화에 유리합니다.
- **스타일링**: **TailwindCSS**
  - 유틸리티 클래스 기반으로 빠르고 일관된 스타일링을 적용합니다. 
- **제안하는 플러그인/라이브러리**:
  - `@tailwindcss/typography`: 워드프레스와 같은 훌륭한 타이포그래피(글의 줄간격, 폰트 크기 비율 등 가독성)를 손쉽게 적용하기 위한 Tailwind 공식 플러그인.
  - `lucide-react`: 깔끔하고 모던한 아이콘 제공.

## 3. 페이지 및 콘텐츠 구성 (Page Structure)

`/docs`의 마크다운 원문을 일반적인 웹 기반 기술 경력서 UI로 요약 및 재구성합니다. 단일 페이지(Single Page) 또는 최소한의 라우팅으로 스크롤하며 자연스럽게 읽을 수 있는 구조를 제안합니다.

### 3.1 GNB (Global Navigation Bar)
- 로고 (이름)
- 네비게이션: Home, Experience, Projects

### 3.2 메인 콘텐츠 영역
1. **Hero Section (자기소개)**
   - 임의의 Placeholder 텍스트로 우선 구성 (추후 사용자가 별도 전달 예정).
   - "결제 도메인의 신뢰성을 어떻게 만들었는가"라는 핵심 서사 노출.
2. **Work Experience (경력 요약)**
   - **(주) 씽즈**: 백엔드 개발자 (2년)
   - **교보문고**: 백엔드 개발자 (3년)
3. **Key Case Studies (핵심 성과 상세 - 포트폴리오 영역)**
   - `/docs` 내용들을 분석하여, 각 성과별로 `[요약] - [문제 상황] - [해결 과정(아키텍처, 패턴 적용)] - [결과 및 임팩트]` 구조로 UI 템플릿(카드나 아티클 포맷)에 맞게 텍스트를 재작성하여 배치합니다.
   - 대상 성과:
     1. 주문 완료 처리 리팩토링 (Command/Provider 패턴)
     2. 기프트카드 발급 인프라 구축 (Outbox-inspired Kafka 비동기)
     3. 주문 완료 동시성 제어 (비관적 락)
     4. 회원 인증 일원화 및 캐시 적용

## 4. 예상 작업 단계 (Implementation Plan)

1. **프로젝트 초기화**: Next.js (App Router) + TailwindCSS 기반 프로젝트 생성 (`npx create-next-app`).
2. **레이아웃 및 기초 스타일링**: Tailwind 설정, 폰트(가독성 높은 산세리프) 및 글로벌 레이아웃, GNB 컴포넌트 개발.
3. **콘텐츠 추출 및 UI 데이터화**: `/docs`의 내용들을 분석하여 포트폴리오 컴포넌트에 주입할 수 있는 데이터 구조(문장 다듬기 및 요약)로 정제.
4. **UI 컴포넌트 개발**: Hero, Experience, Case Study Card 등 기술 경력서를 표현할 UI 컴포넌트 구현 및 데이터 바인딩.
5. **폴리싱**: 웹사이트 가독성 점검 및 Vercel 배포 준비.
